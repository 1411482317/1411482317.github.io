数据结构与算法笔记

发表于 2020-04-16 | 阅读次数:
字数统计: 1.5k | 阅读时长 ≈ 5

算法定义：

  • 输入输出： 算法具有0个或多个输入，至少有1一个或多个输出。

  • 有穷性： 执行完有效嘅步骤后，会自动结束。

  • 确定性： 每个步骤都有具体含义，唔会出现二义性。

  • 可行性： 每一步都能通过。

算法设计要求：

  • 正确性： 能够得到问题嘅正确答案。

  • 可读性： 方便阅读。

  • 健壮性： 处理唔合法数据时，可以做出相关处理。

  • 效率高同埋存储量低： 字面意思。

算法效率嘅度量：

  • 事后统计法： 通过设计好嘅程序，用计算机计时器对唔同算法编制程序嘅运行时间进行比较，判断效率（高投入回报几率低）

  • 事前分析估算法： 程序编制前，用统计法对算法进行估算（投入前进行估算）

时间复杂度

常见嘅时间复杂程度排序：

O(1) < O(logn) < O(n) < O(nlogn) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)

大O推导法

  • 常数： 指规定嘅数量与数字。

  • 用常数1取代运行时嘅所有加法常数。

  • 喺修改后嘅运行次数函数中，保留最高阶项。

  • 假如最高阶项存在并且唔喺1，就去除与尼个项相乘嘅常数，结果就系大O阶

常数阶

一共执行佐6条语句，但记作常数1

唔理常数喺几多，都记作O(1)

线性阶

一共执行佐1 + (n+1) + 1 = 2n+3次

单个循环（非嵌套循环），随着n变大，计算时间呈直线增长，时间复杂程度为O(n)。

平方阶

执行循环执行100次，内层循环执行100*100次，即喺n嘅平方。

空间复杂性为O(n^2)

  • 假如嵌套循环有3层，空间复杂性为O(n^3)，以此类推

对数阶

  • log： 对数函数。

    例子： a^n = b（a>0,且a≠1）， n叫做以a为底b的对数,记做n=log(a)b,【a喺下标】，空间复杂度为O(logn)

空间复杂度

算法可以用执行时间换取空间容量（空间变大但运行慢），用空间容量获取更快嘅执行时间（空间变小但运行快）。

抽象数据类型（ADT）

  • 原子性： 最底层类型不可分解，例如整形，浮点型与字符型等。

  • 结构类型： 由n种类型组成嘅类型，可分解。例如数组等。

抽象数据类型格式：

  • 并集： A=AUB，把存在喺集合B中，但唔存在喺集合A中 嘅元素插入到集合A中。忽略相同元素，拷贝自己无嘅元素。

线性表

拥有顺序，存储零个或多个数据元素嘅有限序列。

线性表强调喺有限嘅，空间太大会导致空间浪费，空间太小会容易发生溢出。

链表（无序内存空间）

内存空间唔连续，使用指针指向下一个空间嘅位置。任意位置读取时间复杂度为O(n)，任意位置插入或删除时间复杂度为O(1)

  • 单向链表： 只有next指针指向下一个位置。

  • 双向链表： 拥有prev与next指针，可以指向上一个与下一个位置

优点： 长度可随意改动
缺点： 查找元素速度慢

静态链表

用数组描述的链表叫做静态链表。

使用游标实现法，数据喺连续嘅，通过修改游标实现数据位置修改。游标相当于next指针。

  • 默认游标号为索引加一。

  • 游标0表示数组最后一个有数据元素嘅位置。

  • 数组嘅头部与尾部唔会存放数据。

  • 链表头嘅游标指向数组第一个空闲索引，链表尾嘅游标指向第一个有数据元素嘅索引（链表头）

插入与删除数据时，只需要修改游标

循环链表

链表最后一个节点指向首节点。

连接两个链表只需修改链表A嘅next指针，与链表B嘅next指针。

判断单链表是否有环：

  1. 指针A与B一次走一步，但指针A每次都喺从头开始走。一旦AB指向嘅地址一样，但步数唔同则存在环。

  2. 指针A一次走一步，指针B一次走两步，一旦A等于B则有环
